# Auth0 for AI Agents: Teaser Talk Track

## Purpose and Framing

This is a teaser trailer designed to generate interest and curiosity. It is **not** a step-by-step product demo or a technical deep dive. The full walkthrough video follows this trailer. This talk track speaks directly to developers who have built an AI agent and now need to make it production-ready, positioning Auth0 as the identity platform that treats AI agents as first-class citizens so teams can ship securely in weeks instead of quarters.

## 0:40 to 1:05 | Pillar 2: Asynchronous Authorization

**Spoken Copy:**

Knowing who the user is opens the door. But some actions are too sensitive for agent autonomy. That's where asynchronous authorization comes in.

(pause)

## 1:05 to 1:30 | Pillar 3: Token Vault

**Spoken Copy:**

Even with the right approval, agents shouldn't hold standing privileges. Token Vault keeps credentials ephemeral.

(pause)

When the user wants a calendar reminder, the agent gets a fresh credential just for that action. It uses the credential once, then discards it. APIs stay safer, and operational risk drops.

**Show / B-roll:**

End-user POV recording: user asks agent to add a calendar reminder. Show the reminder appearing on a calendar screen. Cross-fade to Auth0 admin dashboard showing Token Vault configuration and credential lifecycle management (request, issue, revoke pattern).

---

## 1:30 to 1:55 | Pillar 4: Fine-Grained Authorization

**Spoken Copy:**

And every credential needs boundaries. Fine-grained authorization makes sure agents only access what they need.

(pause)

When the user asks for their orders, the agent sees their order history alone. No payment method, no one else's data. Compliance is protected by design, not by hope.

**Show / B-roll:**

End-user POV recording: user asks agent to check their orders. Agent displays only that user's order history (no other customer data visible on screen). Cross-fade to Auth0 admin dashboard showing Fine-Grained Authorization policy configuration, with resource-level permission scopes highlighted.
